진짜 간단한 문제

문제

Russian Code Cup에서는 다양한 문제를 많이 준비했다. 하지만 출제진들은 문제들이 별로 달갑지 않은 것 같다. 더 많은 참가자들이 적어도 한 문제를 풀 수 있도록 하려면 문제가 더 간단해야 한다고 한다.

출제진 총괄은 문제의 정답 코드에 따라 문제를 나눈다. 만약 정답 코드가 분기 명령을 많아야 하나 사용하고 반복 명령을 많아야 둘 사용하거나, 분기 명령을 많아야 둘 사용하고 반복 명령을 많아야 하나 사용한다면 이 문제는 간단하다. 출제진은 n 개의 문제를 정답 코드와 함께 준비했고 검수진 총괄에게 결재를 부탁했다. 총괄은 문제 해결에 필요한 분기와 반복 명령어의 수를 계산했고, 어떤 문제가 간단한지 알아보려고 한다.

n 개의 문제마다 분기 명령의 수와 반복 명령의 수가 주어진다.

각 문제가 간단한지 판단해 주자.

입력

첫째 줄에 문제의 수 n이 주어진다. (1 ≤ n ≤ 121) 다음 n 개의 줄마다 분기 명령의 수 i와 반복 명령의 수 f가 주어진다. (0 ≤ i, f ≤ 10)

출력

각 문제마다 문제가 간단하면 Yes, 아니라면 No를 출력한다.
