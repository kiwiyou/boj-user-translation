grupe

문제

$1$부터 $N$까지의 정수가 임의의 순서로 주어진다.

그룹은 연속된 정수들의 집합으로 이루어진다. 예를 들어 $[2], [4, 5], [3, 5, 6, 4]$는 그룹이지만 $[5, 7, 2], [2, 4, 5]$는 그룹이 아니다. 초기 상태에서는 각 정수가 크기 $1$인 그룹을 형성한다고 가정한다.

가능한 연산은 인접한 두 그룹을 합치는 것 뿐이다. 하지만 이 연산은 두 그룹을 합쳐서 그룹을 만들 수 있을 때만 사용할 수 있다.

$N - 1$번의 연산으로 주어진 정수들을 하나의 그룹으로 만들 수 있는지 확인하는 프로그램을 작성하시오. 가능하다면 합치는 순서를 제시해야 합니다.

예시 (예제 입력 3):

			[6] [3] [2] [1] [4] [5]
			[6] [3] [2 1] [4] [5]
			[6] [3 2 1] [4] [5]
			[6] [3 2 1] [4 5]
			[6] [3 2 1 4 5]
			[6 3 2 1 4 5]

입력

첫째 줄에 정수 $N (1 \le N \le 500 000)$이 주어진다.

둘째 줄에 $N$개의 양의 정수가 공백으로 구분되어 주어진다.

출력

첫째 줄에 하나의 그룹으로 만드는 것이 가능한지 여부를 나타내는 'DA' (가능) 나 'NE' (불가능)를 출력한다.

하나의 그룹으로 만드는 것이 가능하다면 이후 $N - 1$개 줄에 걸쳐서 연산 수행 이후 합쳐진 그룹의 가장 작은 정수와 가장 큰 정수를 출력한다.
