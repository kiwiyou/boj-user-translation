세계에서 가장 빠른 정렬 알고리즘

문제

정렬 알고리즘의 속도를 비교할 때는 빅 오 표기법을 자주 사용한다. N 개의 원소를 정렬할 때 선택 정렬은 O(N2) 시간이 걸리고, 병합 정렬과 같은 비교 기반 정렬은 적당한 가정 하에 O(N log(N)) 시간보다 빠를 수 없음이 증명되어 있다. 반면 버킷 정렬은 O(N) 시간에도 정렬할 수 있다. 이는 버킷 정렬이 원소들의 범위를 N보다 충분히 작다는 가정을 하기 때문이다. 일반적으로 정렬 알고리즘의 속도는 정렬할 원소들의 값들에 대한 가정을 얼마나 할 수 있느냐에 달려 있다.

여기서 소개할 정렬 알고리즘은 세계에서 가장 빠른 정렬 알고리즘으로, 매우 빠르지만 흔히 논의되지는 않는 알고리즘이다. 이 알고리즘은 O(1) 또는 상수 시간만에 정렬할 수 있다. 물론 이 알고리즘은 입력이 빠르게 접근할 수 있는 저장소에 저장되어 있는 배열이며 이미 정렬되어 있다고 가정한다. 이 알고리즘을 구현해 보자.

입력

입력은 여러 개의 테스트 케이스로 이루어져 있다. 각 테스트 케이스의 첫째 줄에는 정렬할 원소의 수 N과 정렬한 원소 ai들이 주어진다. (0 < N ≤ 100; 1 ≤ i ≤ N; 0 ≤ ai ≤ 100000) 입력의 마지막 줄에는 0이 하나 주어지며, 입력의 끝을 나타낸다.

출력

각 i번째 테스트 케이스마다, Case i: Sorting... done!을 출력한다.
